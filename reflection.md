## Week 3 Reflection

1. The most confusing thing about Python compared to JavaScript was how different the syntax feels even when the logic is similar. Indentation matters a lot more, and I had to get used to colons, `None`, and type hints.
2. An HTTP status code tells you whether a request succeeded or failed. For example, `404` means the resource was not found, like when someone asks for a book ID that does not exist.
3. A path parameter is part of the URL path, such as `/books/1`, where `1` is the book ID. A query parameter comes after `?`, such as `/books?status=reading`, and is usually used for filtering.
4. If I restarted the server right now, all the data would disappear because it is only stored in memory. That is a problem because users would lose everything, and later we will fix it by storing data in a real database.
